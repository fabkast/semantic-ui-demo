Características principales

Diseño adaptable

Diseñado completamente con em y rem.

Auto explicativo

Uso de lenguaje simple y común.

Indiferente a etiquetas

Se puede usar cualquier etiqueta con los elementos UI.

Fuerte enfoque al uso de patrones y la reusabilidad

Las definiciones de elementos pueden ser expresadas en grupos y tienen atributos compartidos evitando declaraciones repetitivas.

Portable e independiente

Usa sólo los componentes que necesites.

Javascript

Incluye componentes javascript, librería de animaciones y validación de formularios.

Preprocesadores CSS

Usa LESS y tiene un port oficial a SASS.

Documentación extensiva

Encuéntrala en su sitio: semantic-ui.com

Definitions

Las definiciones (definitions) en Semantic proveen de un formato reusable para compartir un diseño de interfaz.

Se tratan de grupos de css, fuentes, imágenes y javascript que conforman un sólo elemento.

Tipos de definiciones

UI Elements

Bloques básicos. Existen por sí solos o en grupos homogéneos.

Button Divider Header Icon Image Input Label Loader Progress Reveal Segment Step

UI Collections

Grupo heterogéneo de varios componentes que usualmente pueden ser encontrados juntos.

Breadcrumb Form Grid Menu Message Table

UI Views

Una manera de presentar contenido común.

Comment Feed Item List

UI Modules

Elementos que se definen esencialmente por su comportamiento.

Accordion Checkbox Dimmer Dropdown Modal Popup Rating Shape Sidebar Transition

UI Behaviors

Elementos que se definen esencialmente por su comportamiento.

Form Validation
